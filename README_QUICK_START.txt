БЫСТРЫЙ ЗАПУСК ДИПЛОМНОГО ПРОЕКТА НА WINDOWS
==============================================

Проект состоит из трех частей:
1) frontend/ — сайт администрации;
2) backend/ — FastAPI API для обращений;
3) crm/ — CRM-панель сотрудников, открывается через backend по адресу /crm.

Самый простой запуск:
---------------------
1. Распакуйте архив проекта в обычную папку, например:
   C:\diplom_appeals_crm

2. Откройте папку проекта.

3. Дважды нажмите файл:
   start_project.bat

4. Дождитесь установки зависимостей и запуска двух окон.

5. Откройте адреса:
   Сайт: http://localhost:5500
   CRM:  http://localhost:8000/crm
   API:  http://localhost:8000/docs

Демо-вход в CRM:
----------------
Логин: admin
Пароль: admin123

Дополнительные сотрудники для проверки:
---------------------------------------
Логин: manager
Пароль: manager123
Роль: руководитель/менеджер

Логин: employee
Пароль: employee123
Роль: сотрудник

Как проверить работу системы:
-----------------------------
1. Откройте сайт: http://localhost:5500
2. Нажмите кнопку «Помощник».
3. Выберите «Подать обращение».
4. Заполните ФИО, телефон, адрес, категорию, тему и текст.
5. При желании прикрепите фотографию.
6. Согласитесь на обработку персональных данных.
7. Отправьте обращение.
8. Скопируйте номер обращения BR-2026-00001.
9. Откройте CRM: http://localhost:8000/crm
10. Войдите admin / admin123.
11. Откройте обращение, смените статус, назначьте ответственного и добавьте комментарий.
12. На сайте откройте «Статус обращения» и проверьте статус по номеру и телефону.

Если PowerShell запрещает запуск скриптов:
------------------------------------------
Используйте именно .bat-файлы. Они запускаются через обычный CMD и не требуют изменения политики PowerShell.

Если порт занят:
----------------
Backend использует порт 8000.
Frontend использует порт 5500.
Если один из портов занят, закройте старые окна CMD или перезагрузите компьютер.

Важно для диплома:
------------------
Локально база создается автоматически в backend/appeals.db.
Для размещения на Render лучше использовать PostgreSQL и переменную DATABASE_URL.
