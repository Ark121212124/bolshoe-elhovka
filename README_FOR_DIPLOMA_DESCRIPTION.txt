КРАТКОЕ ОПИСАНИЕ ДЛЯ ДИПЛОМА
============================

В дипломном проекте реализована информационная система приема и обработки обращений граждан.
Система построена по раздельной архитектуре:

- пользовательский сайт администрации работает как frontend;
- обращения отправляются через REST API;
- backend реализован на FastAPI;
- данные сохраняются в PostgreSQL;
- сотрудники администрации работают с обращениями через CRM-панель;
- к обращениям можно прикреплять фотографии;
- в системе реализованы статусы, назначение ответственного, комментарии и история действий.

Основной сценарий работы:
1. Гражданин открывает сайт администрации.
2. Через помощника заполняет обращение: ФИО, телефон, адрес, категорию, тему, текст и фотографию.
3. Обращение передается в backend API и сохраняется в базе данных.
4. Сотрудник открывает CRM-панель и видит новое обращение.
5. Руководитель назначает ответственного сотрудника.
6. Сотрудник меняет статус и добавляет комментарии.
7. Гражданин проверяет статус обращения на сайте по номеру и телефону.

Стек реализации:
- HTML, CSS, JavaScript — frontend сайта;
- FastAPI — backend API;
- SQLAlchemy — работа с базой данных;
- PostgreSQL — хранение данных;
- Render.com — размещение frontend, backend и базы данных;
- GitHub — хранение исходного кода и автоматический деплой.
