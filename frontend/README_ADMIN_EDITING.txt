Что добавлено:
1. После входа в админку на всех страницах сайта появляется плавающий блок «Админ-редактор».
2. Нажмите «Включить правку».
3. На странице можно нажимать на текст, заголовки, кнопки, ссылки, фотографии и иконки.
4. После нажатия откроется окно редактирования.
5. Изменения сохраняются в браузере и сразу применяются на сайте.

Фотографии:
- Исходные фото и иконки в папках сайта НЕ изменялись.
- В редакторе можно указать новый путь к изображению, например:
  assets/img/custom/hero-main.jpg
  assets/img/news/news-01.jpg
  assets/img/icons/site-logo.png
- Также можно выбрать файл с компьютера. В демонстрационной версии он сохранится в браузере.


Размер текста и фотографий:
- При редактировании текста можно менять размер шрифта и межстрочный интервал.
- При редактировании кнопки или ссылки можно менять размер текста.
- При редактировании фото/иконки можно менять ширину, высоту, подгонку изображения и позицию.
- Размер можно писать так: 18, 18px, 1.2rem, 100%, auto. Если указать просто число, сайт сам добавит px.
- Галочка «Сбросить размер» возвращает стандартный размер элемента.

Перенос правок:
- В блоке «Админ-редактор» есть кнопка «Скачать правки JSON».
- Этот файл можно импортировать через кнопку «Импорт JSON».

Важно:
- Это демонстрационный статический сайт без сервера и базы данных.
- Для настоящего рабочего сайта нужно подключить серверную админку, базу данных, авторизацию и загрузку файлов на хостинг.

ДОПОЛНИТЕЛЬНО: ПЕРЕМЕЩЕНИЕ БЛОКОВ
1. Войдите в админку.
2. Откройте нужную страницу сайта.
3. В плавающем блоке «Админ-редактор» нажмите «Включить перемещение».
4. Нажмите на карточку, фотографию, текстовый блок или секцию.
5. В окне можно:
   - изменить порядок блока в сетке;
   - сдвинуть блок вверх/вниз или влево/вправо;
   - нажать «Перетащить мышью» и вручную передвинуть выбранный блок.
6. Нажмите «Сохранить положение». Правки сохраняются в браузере и входят в экспорт JSON.

Важно: фотографии и файлы изображений при этом не изменяются. Меняется только положение элементов на странице.

ДОБАВЛЕНИЕ И УДАЛЕНИЕ БЛОКОВ

В этой версии админ-редактор дополнен управлением блоками страницы.

1. Чтобы добавить новый блок:
   - войдите в админку;
   - откройте нужную страницу сайта;
   - в панели «Админ-редактор» нажмите «Добавить блок»;
   - выберите тип блока: карточка, текстовый блок, фото-блок или контактная карточка;
   - заполните заголовок, текст, контакты, путь к фотографии и ссылку кнопки;
   - нажмите «Добавить блок».

2. Чтобы добавить блок после конкретной карточки:
   - включите «Включить перемещение»;
   - нажмите на нужную карточку или блок;
   - в окне выберите «Добавить после»;
   - заполните данные нового блока.

3. Чтобы удалить ненужный блок:
   - включите «Включить перемещение»;
   - нажмите на блок, который нужно убрать;
   - нажмите «Удалить блок».

4. Чтобы вернуть удалённые блоки:
   - нажмите «Вернуть удалённые» в панели «Админ-редактор».

5. Важно:
   - изменения сохраняются в браузере;
   - для переноса изменений на другой компьютер или перед размещением сайта нажмите «Скачать правки JSON»;
   - на другом устройстве правки можно загрузить через «Импорт JSON».

Фотографии сайта не заменяются автоматически. При добавлении блока можно указать путь к уже существующему изображению или выбрать файл с компьютера. При выборе файла он сохраняется как локальная правка в браузере.

УДАЛЕНИЕ ОТДЕЛЬНЫХ ТЕКСТОВ, ССЫЛОК И СТРОК

В новой версии можно удалять не только крупные карточки, но и отдельные элементы внутри блока:
- отдельный текст;
- кнопку;
- ссылку;
- строку материала;
- фотографию или иконку.

Как удалить строку материала или ссылку:
1. Войдите в админку.
2. Откройте нужную страницу.
3. Нажмите «Включить правку».
4. Кликните по нужной строке, кнопке или ссылке.
5. В открывшемся окне нажмите «Удалить эту кнопку/ссылку» или «Удалить этот текст».
6. Если нужно вернуть удалённое, нажмите «Вернуть удалённые» в панели администратора.

Так можно убрать лишние пункты вроде «Сведения по электроэнергетике», «Материалы по ...», «Открыть материал / официальный источник» и другие элементы внутри карточек.


ПОСТОЯННОЕ СОХРАНЕНИЕ ПРАВОК ДЛЯ GITHUB / RENDER

Чтобы правки, сделанные через админ-редактор, были видны не только в вашем браузере, но и после загрузки сайта на GitHub и Render:

1. В админ-редакторе нажмите «Скачать правки JSON».
2. Скачанный файл должен называться site-edits.json. Если браузер добавил номер, переименуйте его именно в site-edits.json.
3. Поместите этот файл в папку:
   assets/data/site-edits.json
4. Загрузите обновлённый проект на GitHub.
5. Render при новом деплое заберёт этот JSON и применит правки автоматически для всех посетителей сайта.

Важно:
- Если вы меняли путь к фотографии на уже существующий файл, убедитесь, что этот файл тоже лежит в папке проекта и загружен на GitHub.
- Если вы выбирали фото с компьютера через админку, оно сохранялось только в браузере. Для постоянного результата положите фото в папку assets/img/... и в редакторе укажите путь к нему.
- При локальном открытии через file:// браузер может не загрузить assets/data/site-edits.json. На Render и GitHub Pages загрузка работает нормально.
